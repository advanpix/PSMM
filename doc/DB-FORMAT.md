# AllKnownAdvanpix file format

Authoritative reference for the on-disk layout of `AllKnownAdvanpix`
(and any file in the same family, including the per-degree
`roots/deg-NNNN.txt` blocks that re-use the same header line).

## Line types

The file is plain text, one record per line, with one of two kinds of
line:

1. **Comment / header** — starts with `#`. Used to group entries by
   degree, e.g. `# degree = 8, 1 polynomials`. Generated by
   `psmm -merge` between blocks of differing degree. Readers ignore
   these lines.
2. **Data row** — describes one primitive irreducible reciprocal
   polynomial with Mahler measure $M(P) < 1.3$. Fields are
   whitespace-separated.

Blank lines may appear and are ignored.

## Data-row field layout

```
N M NNZ H L K U Q R c_0 c_1 ... c_{N/2}
```

| Field | Type | Meaning |
|---|---|---|
| `N` | int | polynomial degree (even) |
| `M` | decimal string | Mahler measure, **exactly 72 fractional digits**, leading `1.`, rounded with `ROUND_HALF_EVEN` |
| `NNZ` | int | non-zero half-coefficients, excluding the leading `c_0 = 1` |
| `H` | int | height — `max\|c_k\|` over the full polynomial |
| `L` | int | length — `sum\|c_k\|` over the full polynomial |
| `K` | int | count of roots strictly outside the unit disk |
| `U` | int | count of roots on the unit circle |
| `Q` | int | count of complex non-unity roots |
| `R` | int | count of real non-unity roots |
| `c_0` .. `c_{N/2}` | int | half-coefficients, integer-valued; `c_0` is the **leading** coefficient (i.e. coefficient of `x^N`), and `c_0 = 1` always |

Reciprocity identities that every well-formed row satisfies:
`2K + U = N` and `Q + R = 2K`.

The polynomial is reciprocal (palindromic): `c_k = c_{N-k}` for all `k`,
so storing the half-coefficients is sufficient.

## Mahler measure precision

The 72-digit choice is fixed; tools writing entries must round (not
truncate) their high-precision computation down to exactly 72 fractional
digits. Storing more digits than 72 is a format violation and
inconsistent with the rest of the file. Storing fewer (e.g. trailing
zeros from a low-precision compute) silently corrupts downstream
analyses.

`psmm.cpp` uses GMP's `mpf2string(..., 72)` to produce this format with
half-even rounding. Python tools should match — see e.g.
`tools/recompute_pari_mahler.py` which calls `decimal.Decimal.quantize`
with `ROUND_HALF_EVEN`.

## Worked example

```
 10 1.176280818259917506544070338474035050693415806564695259830106347029688377 4 1 9 1 8 0 2 1 1 0 -1 -1 -1
```

decoded:
- `N = 10`
- `M = 1.176280818259917506544070338474035050693415806564695259830106347029688377`  (Lehmer's number, 72 fractional digits)
- `NNZ = 4`, `H = 1`, `L = 9`
- `K = 1`, `U = 8`, `Q = 0`, `R = 2`  ✓ `2·1 + 8 = 10`, `0 + 2 = 2·1`
- half-coefficients `[1, 1, 0, -1, -1, -1]`, full polynomial:
  `x^10 + x^9 - x^7 - x^6 - x^5 - x^4 - x^3 + x + 1` (Lehmer's polynomial)

## Sort order

`psmm -merge` writes the file sorted by `(N ascending, M ascending)`.
Tools that produce delta files for merging do not need to pre-sort —
the merge step re-sorts the combined output.

## Tools that read or write this format

- `psmm` (`-merge`, `-known`, `-addto`, `-analyze`) — canonical
- `tools/bulk_verify.py` — read only
- `tools/extract_parametric_finds.py` — write (delta files)
- `tools/scan_pn_convergence.py` — write (delta files, via `--db-output`)
- `tools/recompute_pari_mahler.py` — read + in-place patch (rounds M)
- `tools/verify_db_uniqueness.py` — read only
- `tools/compute_roots.py` — read, emit `roots/deg-NNNN.txt` blocks
  whose header is the original data row verbatim
