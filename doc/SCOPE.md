# AllKnownAdvanpix ŌĆö scope

This file documents what `AllKnownAdvanpix` does and does *not* contain.
The DB itself does not carry header comments because `psmm -merge`
strips them on every merge pass.

## File format

Each row is one polynomial in space-separated columns:

```
<N> <M> <NNZ> <H> <L> <K> <U> <Q> <R> <c_0> <c_1> ... <c_{N/2}>
```

where the columns are:

| Column | Meaning |
|---|---|
| `N` | Polynomial degree (always even ŌĆö reciprocal polynomials of odd degree have $x = -1$ as a root and reduce to the even-degree case). |
| `M` | Mahler measure $M(P) = \|a_N\| \prod_{\|z_i\|>1} \|z_i\|$, stored at **72 fractional digits** precision, ROUND_HALF_EVEN. |
| `NNZ` | Number of nonzero coefficients among $a_1, a_2, \ldots, a_{N/2}$ (the half-coefficients, excluding the fixed $a_0 = 1$). |
| `H` | Height $\max_k \|a_k\|$. |
| `L` | Length $\sum_k \|a_k\|$ over the full polynomial. |
| `K` | Count of roots strictly outside the unit disk. |
| `U` | Count of roots on the unit circle (even ŌĆö they come in complex-conjugate pairs). |
| `Q` | Count of complex non-unity roots (multiple of 4 ŌĆö Salem quadruplets $\{z, \bar z, 1/z, 1/\bar z\}$). |
| `R` | Count of real non-unity roots (even ŌĆö reciprocal pairs $\{z, 1/z\}$). |
| `c_0 .. c_{N/2}` | Half-coefficients; full polynomial recovered by reciprocity $a_k = a_{N-k}$. |

Inline `# degree = N, K polynomials` headers separate per-degree
sections ŌĆö these are auto-generated by `psmm -merge` and kept current
across merges.

## What's in the DB

The project name says "small" Mahler measure, not "below 1.3". The 1.3
threshold was a historical convenience matching Mossinghoff's
[Known180](../Known180) list. The DB has now been extended past it:

| M region | What we have | Source |
|---|---|---|
| $M < 1.3$ | **Effectively exhaustive** within the brute-force search space ($H \le 1$, $\mathrm{NNZ} \le 3$, $N \le 456$) plus analytical extensions of identified parametric families to $N \le 1500$. | PSMM brute force + parametric scans for $C[2,3,\pm]$ ("Max-U" / 7-term) and $S5[\Phi_3]$, $S5[\Phi_6]$ (5-term reciprocal Salem). |
| $M \in [1.3, 1.5]$ | **Opportunistically catalogued**, NOT exhaustive. Currently a handful of entries that arose as non-cyclotomic factors of reducible 5-term polynomials. | `tools/factor_5term_reducibles.py` backfill. |
| $M \ge 1.5$ | **Not in DB.** No scan has logged this region. Future scans may relax their upper threshold. | ŌĆö |

## What's *not* in the DB

- **Reducible polynomials.** Every entry is irreducible over $\mathbb{Z}$.
- **Non-reciprocal polynomials.** The half-coefficient storage assumes
  $a_k = a_{N-k}$.
- **Odd-degree polynomials.** Reciprocal polynomials of odd degree
  factor as $(x + 1) \cdot R(x)$ with $\deg R$ even; only $R$ is stored.
- **`x ŌåÆ -x` and other equivalence-class twins.** `psmm -merge`
  canonicalises by some equivalence (at minimum $x \to -x$); only one
  representative per class is stored.

## Comparing to Known180

Known180 has 8,438 polynomials, all with $N \le 180$ and $M < 1.3$.
For an apples-to-apples comparison, restrict to the same filter:
`awk '$1 <= 180 && $2 < 1.3'` on `AllKnownAdvanpix`. The DB contains
all Known180 entries plus 96 additional $N \le 180$ entries that the
old Known180 missed (verified during the original PSMM ingest).

## How to filter

Any tool can filter on $M$ at read time. Standard one-liners:

```sh
# DB rows with M < 1.3 (the "classical" subset)
awk '$1 ~ /^[0-9]+$/ && $2 + 0 < 1.3' AllKnownAdvanpix

# DB rows with M in [1.3, 1.5)
awk '$1 ~ /^[0-9]+$/ && $2 + 0 >= 1.3 && $2 + 0 < 1.5' AllKnownAdvanpix
```

`psmm -analyze` and most plot tools have an optional `--m-max` or
similar parameter ŌĆö check each tool's `--help`.

## Why no hard threshold in the DB

The "small Mahler measure" research program is interested in:

1. The Lehmer region $M$ slightly above 1.
2. The accumulation structure (Boyd-Lawton convergence loci) of
   families.
3. The transition from "small" to "ordinary" as $M$ grows.

A hard cutoff at 1.3 prevents exploring (3) and limits (2) artificially.
The current DB is the union of (a) the exhaustive sub-1.3 catalogue and
(b) whatever additional data scans happen to surface above 1.3. Future
scans should follow the
[scan-completeness rules](../tools/factor_5term_reducibles.py) ŌĆö log
everything above a wider threshold (default 1.5), gate the DB-merge by
the project's current focus threshold (1.3 today).
